# 智能送货小车系统 — 开发报告

## 项目概况

| 项目 | 内容 |
|------|------|
| 名称 | 智能送货小车系统 |
| 周期 | 2 周 |
| 仓库 | github.com/swwtat/CarApp |

## 技术选型

| 模块 | 技术栈 |
|------|--------|
| Web 后台 | Node.js + Express + SQLite (sql.js) |
| Web 前端 | 原生 HTML/CSS/JS，无框架 |
| Android APP | Kotlin + Jetpack Compose + Material3 |
| 小车底层 | ROS2 Humble + Navigation2 + Cartographer SLAM |
| 小车感知 | RPLIDAR A1 激光雷达 + Astra Pro Plus 深度相机 |
| 人脸识别 | MTCNN 检测 + SFace (FaceNet ONNX) 嵌入 + 余弦比对 |
| 通信协议 | 自定义 TCP 帧（`$01<type><size><data><checksum>#`）|

## 项目结构

```
CarApp/
├── app/                  Android APP
├── web-admin/            Web 后台管理端
├── icar_face/            小车智能配送 ROS2 包
├── face_recognition/     人脸识别训练 & 数据
└── yolo_training/        YOLO 交通标志检测训练
```

## 各模块完成情况

### Web 后台

- RESTful API：订单 CRUD、收件人管理、小车配置、统计
- TCP 订单推送：自定义帧编码，type=20/21
- 配送状态跟踪：pending → assigned → navigating → scanning → delivered
- 教室楼层可视化
- 人脸照片自动匹配与嵌入

### Android APP

- TCP 遥控：摇杆 + D-pad 双模式
- 摄像头 MJPEG 实时预览
- 单帧拍照保存
- 网络连接管理 + 心跳保活

### 小车配送系统 (icar_face)

- 配送调度引擎：状态机编排全流程
- Nav2 自主导航集成：NavigateToPose Action
- 人脸检测 + 识别管线：MTCNN → SFace → 余弦比对
- 激光雷达避障警卫
- 语音状态播报
- 轻量级桥接脚本（subprocess 方案，零 Python 依赖）

### 人脸识别

- 注册用户嵌入向量预计算
- ONNX 模型导出（93.9 MB → 0.7 MB）
- 最优阈值标定（0.64，TAR=0.90，FAR=0.0001）
- 陌生人数据集下载与评估

## 关键设计决策

1. **TCP 自定义协议**：统一 Android APP 和 Web 后台与小车通信，帧格式简洁、校验可靠
2. **独立端口隔离**：摇杆遥控用 6000，订单下发用 6001，互不干扰
3. **容器内 ROS2**：Nav2 跑在 Docker 容器中，桥接脚本可选容器内/外两种模式
4. **人脸数据内嵌**：订单 JSON 中直接携带 base64 人脸照片，小车无需额外网络请求
