# 智能送货小车系统 — 需求文档

## 项目背景

在校园/教学楼场景下，利用智能小车实现快递自动配送，减少人工派送成本。

## 系统目标

Web 端接收配送订单 → 小车自动导航到目标教室 → 人脸核验收件人身份 → 交付货物 → 返回起点。

## 功能需求

### Web 后台管理端

| 编号 | 功能 | 描述 |
|------|------|------|
| W1 | 订单管理 | 创建、查看、修改、删除配送订单 |
| W2 | 收件人管理 | 录入收件人信息及人脸照片 |
| W3 | 教室管理 | 配置楼层教室分布 |
| W4 | 小车管理 | 配置小车 IP/端口，查看状态 |
| W5 | 订单下发 | 通过 TCP 将订单推送至小车 |
| W6 | 数据看板 | 订单统计概览 |

### Android 手机 APP

| 编号 | 功能 | 描述 |
|------|------|------|
| A1 | 摇杆遥控 | 模拟摇杆控制小车移动 |
| A2 | 方向键遥控 | D-pad 按钮式方向控制 |
| A3 | 摄像头预览 | 实时查看小车摄像头画面 |
| A4 | 拍照 | 抓取单帧保存到相册 |

### 小车端

| 编号 | 功能 | 描述 |
|------|------|------|
| C1 | 订单接收 | TCP 监听接收 Web 后台下发的配送订单 |
| C2 | 自主导航 | 基于 ROS2 Nav2 + LiDAR SLAM 自动导航至目标 |
| C3 | 实时避障 | 激光雷达检测障碍物并自动绕行 |
| C4 | 人脸核验 | 到达后拍照，与收件人人脸比对 |
| C5 | 状态上报 | 配送进度实时回传 Web 后台 |
| C6 | 返回起点 | 配送完成后自动返回充电/等待位 |

## 技术指标

| 指标 | 要求 |
|------|------|
| 配送范围 | 单楼层（逸夫楼 5 层，501-512 教室）|
| 导航精度 | < 0.3m（到达容差）|
| 人脸核验准确率 | TAR > 90%，FAR < 0.1% |
| 避障响应 | 实时避障，支持动态障碍物 |
| 通信方式 | WiFi TCP + HTTP |

## 系统架构

```
Web 后台 (Express) ──TCP──▶ 小车 (Jetson + ROS2)
                                │
                                ├── Nav2 自主导航
                                ├── LiDAR 避障
                                ├── 人脸核验
                                └── 状态上报 (HTTP)

Android APP ──TCP──▶ 小车 (摇杆遥控)
```
